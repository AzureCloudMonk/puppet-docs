すべてのモジュールには、1つまたは複数のクラスが含まれています。
クラスとは、Puppetコードを1つにまとめて名前を付けたもので、Puppet Enterpriseでノードを設定して管理する際の基本的な単位です。

モージュールインストールクイックスタートガイドでインストールした puppetlabs-apacheモジュールには、apacheと呼ばれるクラスが含まれています。
このクイックスタートガイドでは、apacheクラスをagentノードに適用します。これにより、デフォルトのApache仮想ホストをagentノードで立ち上げられるようになります。

前提条件: このガイドでは、すでにPEがモノリシックインストールされており、少なくとも1つの*nix agentノードとpuppetlabs-apacheモジュールがインストールされているものと想定します。

ステップ1:新しいノードグループを作成する

  1. PEコンソールで、[Nodes] > [Classification]の順にクリックし、[Add group]をクリックします。
  2. 新規ノードグループのオプションを指定します。
    • Parent name – [All Nodes]を選択します。
    • Group name – apache_exampleなどのように、このノードグループの役割を説明した名前を入力します。
    • Environment – [production]を選択します。
    • Environment group – このオプションは選択しないでください。
  3. [Add]をクリックします。

ステップ2: 新しいノードグループにノードを追加する

ノードグループにノードを追加するには、グループに含めるノードを定義するルールを作成します。
  1. apache_exampleグループをクリックします。
  2. RulesタブのCertnameエリアのNode nameフィールドに、PE agentノードの名前を入力します。
  3. Pin nodeをクリックします。
  4. [Commit changes]をクリックします。
  5. ほかのノードも追加したい場合は、ステップ5～7を繰り返します。

  • 複数のノードを同時にグループに追加することはできますか？

ステップ3: apacheクラスをexampleグループに追加する

ノードグループが作成されましたので、次にこのノードグループへ適切なクラスを追加します。

  1. コンソールで別の場所へナビゲートされていない限り、 Classificationページには、まだapache exampleノードグループが表示されているはずです。
    Classタブで、Class nameフィールドに「apache」と途中まで入力するだけでオートコンプリートリストに表示されますので、クリックして選択します。

  2. Add classをクリックし、Commitボタンをクリックします。

これで、apacheクラスがagentのクラスリストに表示されます。
このリストは、Nodes > Inventoryの順にクリックし、Inventoryリストのノードをクリックすることで見ることができます。
ノード詳細のページが表示されたら、[Classes]タブをクリックします。

ステップ4：変更を適用し、実行中のクラスを確認する

  1. agentのコマンドラインから、puppet agent -tを実行します。
    次に、/var/www/html/ディレクトリに移動し、index.htmlという名前のファイルを作成します。

  2. 任意のテキストエディタでindex.htmlを開き、なんらかの内容( 「Hello, World!」など)を追加します。

  3. agentのコマンドラインから、puppet agent -tを実行します。
    これで、新たに割り当てたクラスが各ノードに対して反映されます。

  4. 1～2分待ちます。

  5. ブラウザを開き、agentノードのIPアドレスを入力し、最後にport 80を付け足します(例：http://myagentnodeIP:80/)。
    /var/www/html/index.htmlの内容が表示されます。

ステップ5:コンソールでクラスパラメータを編集する

コンソールを使えば、モジュールコードを直接編集せずに、クラスパラメータの値を修正することができます。
  1. コンソールで[Nodes] > [Classification]の順にクリックし、apache_exampleグループを探して選択します。
  2. Classesタブをクリックし、クラスリストでapacheを探します。

  3. Parameter Nameドロップダウンリストから、編集したいパラメータを選択します。
    この例では、docrootを選択します。

    注意: 一部のパラメータについては、値がグレーのテキストで表示されますが、これはデフォルト値です。この値としては、リテラル値かPuppet変数のいずれかを使えます。
    パラメータの追加後にDiscard changesを選択すれば、この値を元に戻すことができます。
  4. Valueフィールドに/var/wwwと入力します。
  5. Add parameterをクリックし、Commitボタンをクリックします。

  6. Puppet Enterpriseが管理しているノードのコマンドライン上でもpuppet agent -tを実行します。

    これで、Puppetが実行され、Puppet Enterpriseにより新しい設定が作成されます。

Apache Webサーバーのルートディレクトリが、デフォルトの/var/www/htmlに代わり、/var/wwwが設定されました。
Webブラウザでhttp://myagentnodeIP:80/を再読み込みすると、/var/wwwディレクトリのファイルリストが表示されます。
htmlをクリックすると、ブラウザで/var/www/html/index.htmlの内容が再び表示されます。

これで、Puppet EnterpriseがagentノードでデフォルトのApache仮想ホストを管理するようになっています。
