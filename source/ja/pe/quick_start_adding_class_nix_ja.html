概要

すべてのモジュールには、1つまたは複数のクラスが含まれています。 クラスとは、Puppetのコードを1つにまとめて名前を付けたもので、Puppet Enterpriseにおいてノードを設定し管理する主な方法でもあります。 モージュールインストールQSGでインストールした puppetlabs-apacheモジュールには、apacheと呼ばれるクラスが含まれています。 この演習では、apacheクラスを使ってデフォルトのApache仮想ホストを立ち上げます。 「apache_example」というグループを作成し、apacheクラスを追加します。

前提条件: このガイドでは、すでにPEがモノリシックインストールされており、少なくとも1つの*nix agentノードとpuppetlabs-apacheモジュールがインストールされているものと想定します。

apache_exampleグループを作成する

  1. コンソールでNodes>Classificationの順にクリックします。
  2. Node group nameフィールドに「apache_example」と入力し、グループ名を指定します。
  3. Add groupをクリックします。
  4. apache_exampleグループをクリックします。

  5. RulesタブのCertnameエリアのNode nameフィールドに、このグループに追加したいPEで管理されているノードの名前を入力します。

    注意: Parent nameとEnvironmentの値はデフォルト値(All Nodes、production)のままにしておきます。

  6. Pin nodeをクリックします。
  7. Commit 1 changeをクリックします。

  8. ほかのノードも追加したい場合は、ステップ5～7を繰り返します。

    ヒント: 各ノードをピン止めしてノードをグループに追加する方法は、「静的な」ノード追加と表現されます。このQSGの範囲からは外れますが、さらに配備が複雑になる場合はノードのfactsをもとにルールを設定し、ノードを「動的に」追加することもできます。

apacheクラスをExampleグループに追加する

コンソールで別の場所へナビゲートされていない限り、 Classificationページには、まだapache exampleノードグループが表示されているはずです。

apacheクラスをexampleグループに追加する手順:

  1. Classesタブをクリックします。

  2. Class nameフィールドに「apache」と途中まで入力するだけでオートコンプリートリストに表示されますので、クリックして選択します。

  3. Add classをクリックし、Commit 1 changeをクリックします。

    これで、apacheクラスがagentのクラスリストに表示されます。 このリストは、Nodes > Inventoryの順にクリックし、Inventoryリストのノードをクリックすることで見ることができます。 ノードの詳細を表示するページが開きます。

  4. agentのコマンドラインから、/var/www/html/を探します。すでに存在しない場合は、index.htmlと名前をつけたファイルを作成します。

  5. 任意のテキストエディタでindex.htmlを開き、なんらかの内容 (「Hello World」など)を入力するか、すでにあるものを編集します。

  6. Puppet Enterpriseが管理している各ノードそれぞれのコマンドライン上でもpuppet agent -tを実行します。

    これで、新たに割り当てたクラスが各ノードに対して反映されます。 1～2分待ちます。

  7. ブラウザを開き、agentノードのIPアドレスを入力し、最後にport 80を付け足します(例：http://myagentnodeIP:80/)。

    /var/www/html/index.htmlの内容が表示されるはずです。

これで、Puppet EnterpriseがagentノードでデフォルトのApache仮想ホストを管理するようになっています。 Apacheインスタンス管理のオプションについては、ForgeのApache ReadMeを参照してください。 *Nixモジュール記述QSGでは、Apache仮想ホスト上で動作するWebアプリを管理する独自のクラスの記述方法について説明しています。

コンソールでクラスパラメータを編集する

コンソールを使えば、モジュールコードを直接編集せずに、クラスパラメータの値を修正することができます。

apacheクラスのパラメータを編集する手順:

  1. コンソールでNodes>Classificationの順にクリックします。
  2. apache_exampleグループをクリックします。

  3. Classesタブをクリックし、クラスリストでapacheを探します。

  4. Parameter Nameドロップダウンリストから、編集したいパラメータを選択します。 この例では、docrootを使います。

    注意: 一部のパラメータについては、値がグレーのテキストで表示されますが、これはデフォルト値です。この値としては、リテラル値かPuppet変数のいずれかを使えます。 パラメータの追加後にDiscard changesを選択すれば、この値を元に戻すことができます。

  5. Valueフィールドに"/var/www"と入力します。
  6. Add parameterをクリックします。
  7. Commit 1 changeをクリックします。

  8. Puppet Enterpriseが管理している各ノードそれぞれのコマンドライン上でもpuppet agent -tを実行します。

    これで、Puppetが動作し、Puppet Enterpriseにより新しい設定が作成されます。

これで終了です！ Apache Webサーバーのルートディレクトリが、デフォルトの/var/www/htmlに代わり、/var/wwwに設定されました。 Webブラウザでhttp://myagentnodeIP:80/を更新すれば、/var/wwwでファイルのリストが表示されます。 htmlをクリックすると、ブラウザで/var/www/html/index.htmlの内容が再び表示されます。
