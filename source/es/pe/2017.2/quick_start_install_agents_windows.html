Resúmen

Importante: Esta información describe como instalar Puppet agents en nodos Windows. El componente Puppet Master descrito en la "Guía Rápida de instalación de PE" debe ejecutarse en una máquina Linux soportada.

Las siguientes instrucciones de instalación de agents Puppets para ejecutar en nodos versiones soportadas de sistemas operativos Windows. En lo referido Guía Rápida de instalación de agent *nix agent para instrucciones sobre la instalación de agents Puppet en nodos ejecutados bajo un sistema operativo *nix.

Para más información sobre Puppet agents, así como los componentes que componen el despliegue de PE, visite el resumen de la instalación en la documentación PE.

Para más información sobre Windows Puppet agents, incluyendo opciones de instalación adicionales, consultar Instalación Windows Agents.

Prerrequisitos: Esta guía asume que usted ha instalado una implementación PE monolíticas y tiene Puppet master, la consola PE y PuppetDB activado y ejecutándose en un único nodo.

Cómo instalar Windows Puppet Enterprise agents

El instalador PE Windows es un paquete Windows .msi estándar y podrán ejecutar como asistente gráfico.

El instalador debe ejecutarse con privilegios elevados. La instalación de Puppet no requiere reiniciar el sistema.

  1. Descarga y ejecuta el instalador.

  2. Cuando el cuadro de diálogo lo indique, proporcione el hostname de su servidor Puppet master.

  3. Cargue una lista actualizada de nodos pendientes consultando con un click Nodos > Certificados sin firma.

  4. Click Accept All para aprobar las consultas y agregar el nodo.

EL Puppet agent puede ahora recuperar configuraciones del master la próxima ocasión que ejecute Puppet.

Después de la instalación

  • El agent de Puppet correrá como un servicio de WIndows; descarga y aplicará configuración cada 30 minutos (por defecto). Ahora puede asignar clases al nodo de forma normal. VerAsignación de configuraciones a nodos para más detalles. Después de la primera ejecución de Puppet, el servicio MCollective también se ejecutará y el nodo ahora será controlado por MCollective. El servicio de Puppet agent y el servicio MCollective puede ser iniciado y detenido independientemente del uso del administrador de servicios UI o la línea de comando sc.exe. Consulte Ejecutar Puppet en Windows para más detalles.

  • El menú Inicio contendrá una carpeta Puppet, con los enlaces directos para ejecutar Puppet agent manualmente, ejecutando Facter y abriendo una ventana de símbolo de sistema para el uso de Puppet tools. Vea [Ejecutando Puppet en Windows][running] para más detalles.

  • Puppet es agregado automáticamente a la variable de entorno PATH de la máquina. Esto significa que tu puedes abrir cualquier línea de comando y llamar puppet, facter Y los pocos archivos de lote que están en el directorio bin de la instalación de Puppet. Esto también agregará los items necesarios del entorno Puppet en la shell, pero solo durante la ejecución de cada uno de los comandos.
