Referencia de parámetro para pe.conf

La siguiente sección detalla los parámetros que son obligatorios o opcionales en pe.conf cuando instale o actualice.

Un fichero pe.conf es un formato de fichero HOCON que declara parámetros y valores necesarios para instalar y configurar PE.

Los siguientes ejemplos son parámetros válidos y expresiones de valores:

Tipo Value
FQDNs "puppet_enterprise::puppet_master_host": "master.example.com"
Cadenas "console_admin_password": "mypassword"
Boleanos "puppet_enterprise::profile::master::app_management": true
Valores booleanos válidos son true r false (Sensible a mayúsculas y minúsculas, sin comillas). No utilice Yes (y), No (n), 1, o 0.
JSON hashes "puppet_enterprise::profile::orchestrator::java_args": {"Xmx": "256m", "Xms": "256m"}
Integer "puppet_enterprise::profile::console::rbac_session_timeout": "60"

Warning: No utilice comillas simples en los valores de los parámetros. Use comillas dobles como se muestra en los ejemplos.

El ejemplo pe.conf ubicado en el directorio del instalador contiene los parámetros obligatorios y necesarios para una instalación monolítica o dividida. Puede utilizar ese archivo de ejemplo sin necesidad de crear uno propio.

Parámetros para instalaciones monolíticas

Los siguientes parámetros son obligatorios para una instalación monolítica.

Parámetro Value
console_admin_password La contraseña utilizada para iniciar sesión en la consola PE. Por ejemplo, "myconsolepassword"
puppet_enterprise::puppet_master_host El FQDN del nodo que aloja al Puppet master. Por ejemplo, "PUPPET MASTER NODE FQDN"
(En una instalación monolítica, Puede dejar este conjunto en el valor "%{::trusted.certname}".)

Parámetros para instalaciones fragmentadas

Los siguientes parámetros son obligatorios para instalaciones fragmentadas.

Parámetro Value
console_admin_password La contraseña utilizada para iniciar sesión en la consola PE. Por ejemplo, "myconsolepassword"
puppet_enterprise::puppet_master_host El FQDN del nodo que aloja al maestro de marionetas. Por ejemplo, "PUPPET MASTER NODE FQDN"
(En una instalación fragmentada,
no utilice el valor "%{::trusted.certname}" para el Puppet master.)
puppet_enterprise::console_host El FQDN del nodo que aloja la consola PE. Por ejemplo, "PE CONSOLE NODE FQDN"
puppet_enterprise::puppetdb_host El FQDN del nodo que aloja PuppetDB. Por ejemplo, "PUPPETDB NODE FQDN"

Parámetros para configurar y poner a punto Puppet Enterprise

Necesita utilizar los siguientes parámetros para configurar y poner a punto PE según sea necesario.

Parámetros para configurar y poner a punto el master de Puppet

Parámetro Value
pe_install::puppet_master_dnsaltnames Debe sera una cadena. Los DNS altnames serán agregados al certificado SSL generado para el nodo de Puppet master.
Default ["puppet"]si no se especifica ninguno.
puppet_enterprise::profile::certificate_authority Incluya un array de certificados adicionales para poder acceder al /certificate_status
API endpoint. Esta lista es aditiva a la lista de certificados PE de base. Por ejemplo, "examplevm.puppet.com"
puppet_enterprise::profile::master::code_manager_auto_configure Establece a true o false
Si desea o no configurar automáticamente el servicio de gestor de código.
puppet_enterprise::profile::master::r10k_remote Debe ser una cadena.
La URL de git para pasar el archivo r10k.yaml.
Por ejemplo, "git@your.git.server.com:puppet/control.git".
Esto puede ser cualquier URL que es compatible con r10k (y normalmente git). Esta respuesta solo es necesaria
si desea que r10k se configure cuando se instala PE.
puppet_enterprise::profile::master::r10k_private_key Debe ser una cadena.
La ruta del sistema de ficheros local en el Puppet master donde la clave privada SSH puede ser encontrada y utilizada por r10k.
Por ejemplo, "/etc/puppetlabs/puppetserver/ssh/id-control_repo.rsa").
Esta respuesta es solo necesaria si usted quiere configurar 10k configurado cuando se instala PE. Esto debe ser especificado junto a
puppet_enterprise::profile::master::r10k_remote.
puppet_enterprise::profile::master::check_for_updates Establece true orfalse (Default es true.)
Si desea buscar actualizaciones siempre
reinicie el servicio pe-puppetserver. Update checks están activados (true) por defecto.
Consulte desactivar update checkingpara más información.
puppet_enterprise::profile::mcollective::peadmin::mco_loglevel Una cadena que representa la ruta de acceso al archivo de registro del cliente MCollective. Cambie cuando sea necesario.

Parámetros para configuración y puesta a punto de la consola PE y servicios de consola

Parámetro Value
puppet_enterprise::profile::console::classifier_synchronization_period Debe ser un integer. Classifier synchronization period.
Controla cuánto tarda el clasificador de nodos (NC) en recuperar clases
desde el Puppet master. Por defecto son "600" segundos (10 minutos).
puppet_enterprise::api_port El puerto SSL PE sirve el clasificador de nodos. Por defecto "[4433]".
puppet_enterprise::profile::console::rbac_session_timeout Debe ser integer. Especifica cuánto tiempo debe durar la sesión de un usuario, en minutos. Esta sesión es la misma en toda la clasificación de nodos, RBAC y la consola. El valor por defecto es "60".
puppet_enterprise::profile::console::session_maximum_lifetime Debe ser integer. Especifica el período máximo permitido para que una sesión de consola pueda ser válida para. Las unidades soportadas son "s" (segundos), "m" (minutos), "h" (horas), "d" (días), "y" (años). Puede ser fijado en "0" para no expirar antes de la vida máxima del token.
Las unidades se especifican como una sola letra después de un número entero, por ejemplo "1d" (1 día). Si no se especifican unidades, el entero se trata como segundos.
puppet_enterprise::profile::console::console_ssl_listen_port Debe ser integer. El puerto de la consola donde se encuentra disponible.
Por defecto es "[443]".
puppet_enterprise::profile::console::ssl_listen_address Dirección que PE's nginx escucha para la consola PE.
Defecto a "0.0.0.0".
puppet_enterprise::profile::console::classifier_prune_threshold Debe ser intenger. El número de días que debe esperar antes de podar el tamaño de la base de datos del clasificador.
Si establece el valor a "0", El servicio clasificador de nodos nunca reducirá la base de datos.
puppet_enterprise::profile::console::pcp_timeout Un Puppet agent necesita conectar al intermediario PCP para conseguir que Puppet corra vía el botón de la consolaRun Puppet. Establece un integer para especificar cuanto tiempo debería pasar antes que la conexión se cierre.
El botón Run Puppet se establece por defecto en "5" segundos. Si el agente no puede conectar con el intermediario en franja principal, la ejecución expirará.
puppet_enterprise::profile::console::proxy::http_redirect::enable_http_redirect La consola PE redirecciona al HTTPS cuando se intenta conectar con HTTP. Por defecto true. Establece a false para desactivar.
puppet_enterprise::profile::console::classifier_node_check_in_storage El servicio clasificador de nodos puede almacenar un check-in para cada nodo cuando se solicita su clasificación que incluye una explicación de cómo coincide con la regla de cada grupo en el que fue clasificada. Esta deshabilitado por defecto. Por defecto false. Establece truepara activar.
puppet_enterprise::profile::console::display_local_time Por defecto, la consola muestra la fecha/hora en formato UTC (también conocido como Zulu time). Si lo prefiere, puede cambiar la configuración de la consola para mostrar todas las máscara de tiempo en la hora local, con la hora UTC mostrada en hover. Establezca true para mostrar la máscara de tiempo en hora local, con el texto hover mostrando el tiempo UTC o false (por defecto) para mostrar máscara de tiempo en UTC.
puppet_enterprise::console_services::no_longer_reporting_cutoff Determina la cantidad de tiempo que debe pasar después de enviar su último informe antes de que un nodo se considere que no responde. Establezca un entero para especificar el valor en segundos. El valor predeterminado es 3600 segundos.

Parámetros para configurar y puesta a punto los servicios de orquestador y orquestación

Parámetro Value
puppet_enterprise::profile::master::app_management Deshabilitar o habilitar la administración de aplicaciones. Establece true o false.
Activar (true) por defecto.
puppet_enterprise::profile::orchestrator::run_service Deshabilitar o habilitar los servicios de orquestación. Establece a true o false.
Activado (true) por defecto.
puppet_enterprise::profile::agent::pxp_enabled Desactivar el servicio PXP. Establecer true o false.
Si desactiva esta opción, no podrá utilizar el orchestrator o el botón Run Puppet button de la consola.
Activado (true) por defecto.
puppet_enterprise::profile::orchestrator::global_concurrent_compiles Un número entero que determina cuántas peticiones de compilación simultáneas pueden estar pendientes para al Puppet master, en todos los trabajos del orquestador.
El valor por defecto es "8".
puppet_enterprise::profile::orchestrator::pcp_timeout Un Puppet agent necesita conectarse con el intermediario PCP para poder ejecutar Puppet a través del orquestador de Puppet. Establezca un integer para especificar cuánto tiempo debe pasar antes de que la conexión se agote.
The Puppet orchestrator por defecto es "30" segundos. Si el agente no puede conectarse al intermediario en ese marco de tiempo, el tiempo de espera de la ejecución.
puppet_enterprise::pxp_agent::pxp_logfile Una cadena que representa la ruta al archivo de registro del agente PXP. Cambiar según sea necesario. Por defecto, los ficheros logs están en /var/log/puppetlabs/pxp-agent/pxp-agent.log (en *nix) o C:/ProgramData/PuppetLabs/pxp-agent/var/log/pxp-agent.log (en Windows).
puppet_enterprise::pxp_agent::ping_interval Controla la frecuencia con la que los agentes de PXP harán ping a los agentes de PCP. Si los agentes no reciben respuestas, intentarán volver a conectarse.
Por defecto a "120" segundos.

Parámetros para configurar y puesta a punto PuppetDB

Parámetro Value
puppet_enterprise::puppetdb::command_processing_threads Debe ser integer. Define cuantos comandos procesa
PuppetDB Utiliza para clasificar los datos entrantes. Cada hilo puede procesar un comando cada vez.
Este valor predeterminado es la mitad del número de núcleos del sistema. Por ejemplo, "8".
puppet_enterprise::profile::master::puppetdb_report_processor_ensure El Puppet master genera informes de ejecución de agentes cada vez que Puppet se ejecuta y envía
estos a PuppetDB.
Establece a present o absent. Por defecto establecido a present(activado).
puppet_enterprise::puppetdb_port El puerto SSL PuppetDB escucha. Debe ser un entero entre paréntesis.
Por ejemplo, "[8081]".
puppet_enterprise::profile::puppetdb::node_purge_ttl Establece el valor "time-to-live" antes de que PE elimine automáticamente los nodos que han sido
deactivated o expired. Esto también eliminará todos los hechos, catálogos e informes de los nodos relevantes.
Por ejemplo, un valor de "14d" debería definir el tiempo a vivir a los 14 días.
