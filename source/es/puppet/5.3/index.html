Bienvenido al manual de consulta Puppet. Utilice la navegación de la izquierda para moverse.

Cómo funciona

Como introducción a la gestión de sistemas Puppet, consulte Resumen de la arquitectura Puppet.

Comenzemos

Puppet consiste en:

  • Un paquete "All-in-One" puppet-agent que instala Puppet, Ruby, Facter, Hiera y código de soporte.
  • Un paquete puppetserver instala Puppet Server.
  • Un paquetepuppetdb instala PuppetDB.

Para instalar estos, lea las instrucciones de pre-instalación, después, consulte la guías para la instalación de Puppet para Linux, Windows y macOS.

Como moverse

Este manual está dividido en varias secciones, a las que se puede acceder desde la barra lateral izquierda. Algunas páginas importantes:

  • Las release notes cubren las novedades y cambios de esta versión de Puppet.
  • Utiliza los Manuales de Referencia Type para actualizar la información en los conceptos core de Puppet.
  • Puppet utiliza su propio lenguaje de configuración, documentado en esta sección del manual del lenguaje. Usted puede empezar con:
    • El Resumen del Lenguaje, facilita un sumario y un contexto para el lenguaje.
    • El Visual Index, ayuda a encontrar documentos de sintaxis cuando usted sabe lo que parece, pero no sabe como se llama.
  • La guía de Fundamentos de Modules explica como organizar manifiestos Puppet, instalar "modules" pre-construidos desde Puppet Forge, y compartir tus propios "modules".
