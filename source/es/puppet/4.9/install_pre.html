Nota: Este documento cubre los lanzamientos open source de Puppet Para instrucciones de instalación de Puppet Enterprise, revisaresta documentación de instalación.

Antes de instalar Puppet, usted debería realizar las siguientes tareas...

Decidir el tipo de despliegue

Generalmente Puppet utiliza una arquitectura agent/master (cliente/servidor), pero también puede ejecutar una arquitectura autónoma. Su elección determinara los paquetes a instalar y la configuración extra necesaria

Además, Usted debería considerar utilizar PuppetDB, activando más características Puppet y facilitando la consulta y el análisis de los datos que Puppet ofrece sobre su infraestructura.

Aprenda más sobre la infraestructura Puppet aquí.

Servidores designados

Si selecciona la arquitectura estándar agent/master, deberá decidir que servidor o servidores actuarán como Puppet master ( y servidor PuppetDB , si decide utilizarlo).

Usted debería finalizar la instalación y configuración de los servidores Puppet master y PuppetDB, antes de instalar cualquier nodo agent. El master debe correr sobre algún *nix. El master no puede correr sobre máquinas Windows.

El Puppet master debería ser una "maquina dedicada". También debería tener un hostname accesible. Consulta los requerimientos del sistema para los componentes mínimos de hardware.

Nota: Los nodo Agent consultarán por defecto al master con el hostname puppet. Si estás seguro que el hostname apunta al master, puede reducir el tiempo de setup realizando un cambio en la configuración server.

Comprueba la versión de SO y los requerimientos de sistema

Consulta los requerimientos de sistema para la versión de Puppet a instalar, considerando los siguientes puntos:

  • Su Puppet master ha de ser capaz de gestionar el número total de agents a servir.
  • Los SO ofrecen paquetes oficiales para una instalación sencilla.
  • Los SO podrían no proporcionar paquetes Puppet para realizar la instalación. Siempre y cuando la versión de Ruby sea configurable y los requisitos estén instalados ( Consulta la lista de versiones soportadas de Ruby y los requisitos ), deberías realizar una instalación más compleja.

Revisa tu configuración de red.

En despliegues de agent/master, debes preparar tu red para soportar el tráfico de Puppet.

  • Firewalls: El Puppet master debe permitir las conexiones incoming en el puerto 8140, y los nodos agent deben ser capaces de conectar con el master en este puerto.
  • Nombres de dominio: Cada nodo deber tener un hostname único. Forward and reverse DNS debe estar configurado correctamente. (Las instrucciones para la configuración de DNS están fuera de los objetivos de esta guía. Si no dispones de DNS, puede escribir en el fichero /etc/hosts la configuración de cada nodo.)
    • Nota: Por defecto el hostaname de Puppet master es puppet. Los nodos agent pueden estar configurados más rápido, si resuelven este hostname en el Puppet master.

Revisa la zona horaria en el servidor Puppet master

La hora debe estar configurada con exactitud en el Puppet master, por que actuará como entidad certificadora. Te recomendamos el uso de NTP.

(Si la zona horaria es incorrecta, podríamos generar errores entre los agentes, con certificados pasados o futuros. Los cuales tratarían como expirados el resto de nodos. )
